# Überblick

**Ziel:** Studierende praxisnah an saubere Java-Entwicklung heranführen.

## Reihenfolge

1) **Java OOP + Tests** (Domain-Modell, Kapselung, Verantwortlichkeiten)
2) **JSF Web-App** (UI, Managed Beans/CDI, Validierung, Navigation)
3) optional später: Persistence (JPA), REST, ETL, DevOps, …

## Didaktik-Pattern (pro Workshop)

- 20–30 min Input (Konzept)
- 45–60 min Guided Coding (gemeinsam)
- 30–45 min Challenge (Teams/Einzeln)
- 10 min Review (Refactoring / Best Practices)
